# TTS Ident Generator

Windows desktop tool for generating a 16-channel broadcast slate/ident file with:

- a moving video test background
- centered overlay text
- a common spoken phrase at the start on all 16 channels
- grouped per-channel spoken IDs
- a 1 kHz tone on all channels after speech completes

The application uses a Tkinter GUI, Windows `System.Speech` text-to-speech, and FFmpeg for audio/video rendering.

## What it does

This project creates an MXF slate file intended for channel identification and technical checking.

Output behavior:

- Video: `testsrc2` moving test pattern
- Text: large centered overlay, with optional dimmed box behind the text
- Audio: 16 mono PCM tracks at `48 kHz / 24-bit`

Audio timeline:

1. A common spoken phrase plays once at the very start on all 16 channels.
2. Channel IDs play in four simultaneous groups:
   - Group 1: `1, 5, 9, 13`
   - Group 2: `2, 6, 10, 14`
   - Group 3: `3, 7, 11, 15`
   - Group 4: `4, 8, 12, 16`
3. After all spoken IDs finish, a `1 kHz` tone starts on all channels and continues to the end of the clip. To ensure that there is sufficient channel ident and tone, 15 seconds duration is advised.

## Features

- Tkinter GUI for non-command-line use
- FFmpeg auto-detection from `PATH` or common Windows install locations
- Video codec options:
  - `DNxHD`
  - `DNxHR`
  - `ProRes`
- Frame rate options:
  - `50`
  - `59.94`
- Scan options:
  - `Progressive`
  - `Interlaced`
- Field order options for interlaced output:
  - `TFF`
  - `BFF`
- Configurable:
  - output folder
  - base filename
  - overlay text
  - common spoken phrase
  - duration
  - font size
  - text background box
- Live log and encoding progress display
- Reuses generated channel ID WAVs when available

## Requirements

- Windows 11 or Windows with PowerShell and `System.Speech`
- Python 3.10+ recommended
- FFmpeg installed locally, with `ffmpeg.exe` available either:
  - on `PATH`, or
  - at `C:\Program Files\ffmpeg\bin\ffmpeg.exe`, or
  - at `C:\ffmpeg\bin\ffmpeg.exe`

Python standard-library modules only are used for the GUI/application logic. No third-party Python package install is currently required.

## Installation

### 1. Install Python

Install Python 3.10 or newer from the official Python installer for Windows.

During installation, enable the option to add Python to `PATH` if you want to run the script directly from PowerShell or Command Prompt.

### 2. Install FFmpeg

FFmpeg is required. The application does not work without it.

Install FFmpeg on Windows using one of these approaches:

1. Download a Windows build of FFmpeg.
2. Extract it to a local folder such as `C:\ffmpeg`.
3. Confirm the executable exists, for example:

```text
C:\ffmpeg\bin\ffmpeg.exe
```

You can either:

- add the FFmpeg `bin` folder to your system `PATH`, or
- browse to `ffmpeg.exe` inside the application UI

The script will also try these common locations automatically:

- `C:\Program Files\ffmpeg\bin\ffmpeg.exe`
- `C:\ffmpeg\bin\ffmpeg.exe`

To verify the installation, run:

```powershell
ffmpeg -version
```

If that command fails, FFmpeg is not on `PATH`. You can still use the app by setting the full executable path in the `FFmpeg path` field.

### 3. Clone or download this project

Place the project in a local folder on your Windows machine.

### 4. Run the script

From the project directory:

```powershell
python .\Ident_Gen.py
```

 Speech text is common for all channels, for example, the device name/number. It is spoken before the individual channel ident.

## Output

By default, the generated file:

- uses the `.mxf` wrapper
- writes the finished media file to the selected output directory
- writes generated TTS WAV files into a `tts` subfolder inside the output directory

Typical output structure:

```text
output-folder/
  SLATE.mxf
  tts/
    common.wav
    ch01.wav
    ch02.wav
    ...
    ch16.wav
```

## Technical notes

- Video size is fixed at `1920x1080`.
- Audio is encoded as `pcm_s24le`.
- Sample rate is fixed at `48000 Hz`.
- Each audio output is mono.
- Interlaced output is generated by creating field-rate video and then interleaving fields in FFmpeg.
- The default tone level and encode settings are hardcoded in the script.

## Limitations

- Windows-only TTS path: speech generation depends on PowerShell `System.Speech`.
- Font path is currently hardcoded to `C:\Windows\Fonts\arial.ttf`.
- Wrapper extension is currently fixed to `.mxf`.
- Channel ID phrases are generated as `Channel 1.` through `Channel 16.`.
- This is a GUI application; there is no CLI interface at present.

## Troubleshooting

### FFmpeg not found

Set the full path to `ffmpeg.exe` in the GUI if auto-detection fails.

### No speech output

Confirm PowerShell is available and that Windows speech components are installed and functioning.

### FFmpeg encode failure

Check the log window in the application. The script streams FFmpeg output directly so codec, wrapper, or path issues should appear there.
